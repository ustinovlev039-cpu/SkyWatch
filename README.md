# SkyWatch

SkyWatch — консольное Python-приложение для получения и анализа данных о
самолётах над выбранной страной.

Приложение определяет географические границы страны через Nominatim, получает
актуальные данные о воздушных судах из OpenSky Network и преобразует их в
объекты `Aeroplane`. Результаты можно сортировать, фильтровать и сохранять в
JSON-файл.

## Возможности

- получение списка самолётов над выбранной страной;
- сравнение самолётов по высоте и скорости;
- сортировка по высоте и скорости;
- получение топа N самолётов;
- фильтрация по стране регистрации, диапазону высот и статусу на земле;
- расчёт статистики по найденным самолётам;
- сохранение данных в JSON;
- обновление существующей записи по ICAO24;
- удаление самолёта из хранилища;
- обработка ошибок внешних API и некорректных данных.

## Используемые технологии

- Python 3.10+;
- Requests;
- Pytest;
- Flake8;
- Nominatim API;
- OpenSky Network API.

## Установка

Клонируйте репозиторий и перейдите в каталог проекта:

```bash
git clone <адрес-репозитория>
cd SkyWatch
```

Создайте и активируйте виртуальное окружение:

```bash
python -m venv .venv
source .venv/bin/activate
```

Для Windows:

```powershell
.venv\Scripts\activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

После запуска откроется интерактивное меню:

```text
1. Получить самолёты над страной
2. Показать текущий список самолётов
3. Получить топ N самолётов по высоте
4. Найти самолёты по стране регистрации
5. Сохранить текущий список в JSON
6. Показать сохранённые самолёты
7. Удалить самолёт из JSON по ICAO24
0. Выйти
```

Название страны для запроса следует вводить на английском языке. Для работы с
внешними API требуется подключение к интернету.

## Хранение данных

По умолчанию самолёты сохраняются в файл:

```text
data/aeroplanes.json
```

Хранилище создаётся автоматически. При повторном добавлении самолёта с тем же
ICAO24 существующая запись обновляется.

## Структура проекта

```text
SkyWatch/
├── main.py
├── requirements.txt
├── pytest.ini
├── src/
│   └── skywatch/
│       ├── api.py
│       ├── models.py
│       ├── services.py
│       └── storage.py
└── tests/
    ├── test_aeroplane.py
    ├── test_api.py
    ├── test_services.py
    └── test_storage.py
```

- `api.py` — клиенты Nominatim и OpenSky Network;
- `models.py` — модель самолёта и её валидация;
- `services.py` — сортировка, фильтрация и статистика;
- `storage.py` — абстракция хранилища и работа с JSON;
- `main.py` — консольный интерфейс приложения.

## Тестирование

Запуск всех тестов:

```bash
pytest
```

Проверка стиля кода:

```bash
flake8 src tests main.py
```

Тесты не отправляют реальные HTTP-запросы: ответы внешних сервисов заменяются
тестовыми объектами.

## Пример вывода

```text
1. SKW101 | ICAO24: a1b2c3 | Страна: Germany |
   Высота: 10200 м | Скорость: 250.5 м/с | Статус: в воздухе
```

Если отдельные параметры отсутствуют в ответе OpenSky, приложение выводит
`нет данных` и продолжает обработку остальных самолётов.
